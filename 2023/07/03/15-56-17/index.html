MVC三层架构

发表于 2023-07-03 分类于 Java

摘要
本文记录了MVC三层架构的基本思想。

MVC：Model、view、Controller 模型、视图、控制器

  • 早些年，用户直接访问控制层，控制层可以直接操作数据库

MVC三层架构

Model：

  • 业务处理：业务逻辑（Service）
  • 数据持久层：CRUD（Dao）

View：

  • 展示数据
  • 提供链接发起Servlet请求（a，form，img…）

Controller：

  • 接收用户的请求：（requset）
  • 交给业务层处理对应的代码
  • 控制视图的跳转

登录 –> 接收用户的请求 –> 处理请求（获取用户登录的参数）–> 交给业务层处理登录业务–> Dao层查询用户名和密码是否正确 –> 数据库
